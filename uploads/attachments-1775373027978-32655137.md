## 📅 Weekly Project Meeting Logs

### Week 1: Database Setup & Core Authentication
*   **Meeting Focus:** Infrastructure scaffolding and Security
*   **Key Achievements:**
    *   **Database Connectivity:** Successfully connected the application to the PostgreSQL Database using the Prisma ORM.
    *   **Problem Faced & Resolved:** Faced a severe Prisma migration synchronization issue due to conflicting Enum statuses between the User schema and Project schemas. Resolved this by standardizing Prisma Enums (`APPROVED`, `UNVERIFIED`) and wiping the shadow database cache to ensure strict relation mapping.
    *   **Role-Based Access Control:** Implemented a robust authentication and authorization architecture effectively isolating the 3 target user roles: `ADMIN`, `FREELANCER`, and `CLIENT`.
    *   **OTP Security Verification:** Successfully integrated **Nodemailer + Gmail App Password** to automate OTP delivery, preventing unverified ghost accounts from occupying the database.

### Week 2: Profiles, Document Verification, & Initial Project Listings
*   **Meeting Focus:** Identity Verification and Open Market Creation
*   **Key Achievements:**
    *   **Freelancer Profile Creation:** Engineered the dynamic freelancer profile interfaces, supporting multiple skill node inputs and certification uploads.
    *   **PAN/VAT Verification Enforcement:** Secured the bidding engine. Implemented the verification flow so applying for projects is hard-locked until the freelancer uploads PAN/VAT documents and achieves `VERIFIED` status. Built the frontend redirect UI for this barrier.
    *   **Project Creation Engine:** Set up the premium 4-step wizard for clients to post job opportunities.
    *   **Live Marketplace Routing:** Finalized the integration sending successfully posted client projects to the public `/browse-projects` feed, making them visible to the freelancer side.

### Week 3: Project Apply Flows & Proposals Hookup
*   **Meeting Focus:** Bidding Interaction and Application Submission
*   **Key Achievements:**
    *   **Secure Apply Flow:** Finalized the proposal submission ecosystem. Authorized freelancers can now write proposals, view character counts, and apply to listed projects.
    *   **Client Dashboard Sync:** Projects receiving proposals immediately flag state changes, rendering applications within the client's side-by-side review dashboard (`/projects/[id]/applications`).
    *   **Accept Proposal Flow:** Developed the crucial "Accept Proposal" event trigger which updates both proposal and project states simultaneously. 
    *   **Guest Handling:** Established `tryAuthenticate` allowing zero-friction guest browsing, while elegantly forcing a redirect to the login screen only when a guest clicks "Apply".

### Week 4: Contract Generation & Negotiation Cycles
*   **Meeting Focus:** Binding Agreements and Rejection Feedback Loops
*   **Key Achievements:**
    *   **Automated Contract Flow:** Wired the acceptance of a proposal to dynamically generate a draft Contract and automatically jump the client to the `/contracts/[contractId]/edit` workspace.
    *   **Visibility Blackout:** Projects that enter this contract drafting stage are instantly removed from the public marketplace (`status: IN_PROGRESS`) to halt incoming bids.
    *   **Freelancer Approval Loop:** Pushed client-drafted contracts directly to the freelancer for review. 
    *   **Reject & Remark Engine:** Created the revision flow giving freelancers the power to reject a contract with attached "reject remarks," forcing the document back to the client for continuous negotiation until finalized.

### Week 5: Project Management & Milestone Transactions
*   **Meeting Focus:** Task Execution, Delivery, and Payments Tracker
*   **Key Achievements:**
    *   **Active Project Management:** For accepted contracts, a dedicated project management channel (Kanban style) automatically unlocks.
    *   **Kanban Database Linking:** Fully digitized the management flow tying the DND Kanban interface directly to the backend database schemas.
    *   **Milestone Wise Payment Engine:** Architected the logical milestones inside the contract variables so progress on the Kanban board correlates directly to discrete, milestone-wise payment triggers. 
    *   **Current Development Phase:** Focused on ensuring the final handover flow works cleanly from in-progress development straight to project completion and reciprocal reviewing.
